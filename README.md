# 프로젝트 소개 - Dev Dictionary Quiz

개발을 공부하면서 가장 어려웠던 부분 중 하나는 **개념의 휘발성** 이었습니다.
용어나 원리를 이해했다고 생각해도, 시간이 조금만 지나면 기억이 흐릿해져 다시 용어를 찾게 되는 일이 반복됐습니다.
그래서 "개발자 용어를 더 효과적으로 기억하고 복습할 수 있는 방법이 없을까?"라는 고민을 하게 되었습니다. 이를 직접 해결해보고자 **개발자 용어 학습용 퀴즈 애플리케이션**을 만들게 되었습니다.

단순히 랜덤으로 문제를 보여주는 방식이 아니라, 더 나아가 내가 잘 모르는 개념을 더 자주 복습할 수 있는 구조를 직접 설계해보고 싶었습니다. 이를 기반으로 콘솔 환경에서 동작하는 **Dev Dictionary Quiz**를 개발하게 되었습니다.

## 프로젝트 목표

- 개발자 필수 개념을 효율적으로 복습할 수 있는 콘솔 기반 학습 도구 구현
- 사용자의 정답/오답 패턴을 기반으로 약한 개념을 더 자주 출제하는 알고리즘 구현
- Node.js의 파일 입출력을 활용해 학습 이력과 용어 숙련도를 영구적으로 저장
- 3주간 학습한 자바스크립트 문법을 기반으로 활용하여 구현

## 기능 정리

1. 사용자 입력 처리
   - Node.js의 `readline`을 활용해 콘솔에서 사용자의 입력을 받습니다.
   - 숫자 입력 시
     - 숫자가 아닌 입력 -> 재입력 요청
     - 메뉴/보기 범위를 벗어나느 숫자 -> 재입력 요청
2. 카테고리 선택 기반 일반 퀴즈
   - 퀴즈 시작 전에 카테고리 목록을 보여줍니다.
   - 선택한 카테고리에 해당하는 용어만 필터링해서 출제 풀(pool)을 구성합니다.
   - 예시 카테고리: JavaScript, Network, CS, Data Structure, Algorithm, Github, Java
3. 4지선다 퀴즈 모드
   - 하나의 용어를 정답으로 선택하고, 정답 1개 + 오답 3개를 섞어 4지선다 문제를 만듭니다.
   - 문자 형식
     - "이 설명에 해당하는 용어는 무엇일까요?"
     - 1~4번 보기 출력
   - 사용자는 1~4 중 번호를 선택합니다.
   - 즉시 정답/오답 여부와 정답 용어를 출력합니다.
4. 약점 복습 모드
   - 이전 세션에서 틀린 용어들만 모아서 별도의 퀴즈를 진행합니다.
   - `history.json` 에 누적된 `wrongHistory`와 `questionWeights`를 활용해서 틀렸던 문제만 pool로 구성합니다.
     그 pool 안에서도 더 자주 틀린 문제일수록 더 높은 확률로 출제되도록 `weightedRandom()` 알고리즘을 적용했습니다.
5. 파일 입출력 기반 학습 이력 저장
   - `history.json` 에 아래 정보를 저장합니다.
     - 전체 플레이 횟수(playCount)
     - 최고 점수(bestScore)
     - 마지막 점수(lastScore)
     - 틀린 문제 id 기록(wrongHistory)
     - 문제별 가중치(questionWeights)
     - 카테고리별 통계(categoryStats)
6. 가중치 기반 출제 알고리즘
   - `QuestionSelector`는 문제를 선택하는 책임을 가진 모듈입니다.
   - 일반 모드
     - `wrongHistory`에 초함된 용어만 모이서 pool 생성합니다.
     - `questionWeights`를 기반으로 가중 랜덤 선택합니다.
     - 많이 틀릴수록 더 자주 출제됩니다.
7. 학습 기록 초기화 기능
   - 메뉴에서 ** 기록 초기화 **를 선택할 수 있습니다.
     - `history.json`이 초기 상태로 되돌아갑니다.
     - playCount, bestScore, wrongHistory, questionWeights, categoryStats 모두 리셋됩니다.

## 프로젝트 구조

1. `data/` : 정적 데이터와 저장용 JSON
   - `config.json` : 게임 설정값 저장
   - `history.json`: 사용자 학습 히스토리 저장
   - `terms.json` : 실제 문제 데이터
2. `logic/` : 핵심 도메인 로직
   - `Dictionary.js`
     - `terms.json`을 읽어 전체 용어를 메모리에 로딩
     - 카테고리 필터링
   - `QuestionSelector.js`
     - 일반/가중치/카테고리/약점 기반 문제 선택 로직
   - `QuizGenerator.js`
     - 4지선다 퀴즈 한 문제를 생성하는 모듈
   - `Result.js`
     - 한 게임의 결과를 받아 `history.json` 업데이트
   - `Session.js`
     - 한 판의 퀴즈 흐름을 관리하는 클래스
     - 일반 모드/ 약점 모드를 지원
3. `ui/` : 입출력
   - `ConsoleInput` : 사용자 입력
   - `ConsoleOutput` : 결과 출력
   - `MenuView` : 메인메뉴 출력, 카테고리 선택 메뉴
4. `utils/` : 보조 함수, 공통 모듈
   - `FileUtils` : JSON 파일 읽기/쓰기
   - `RandomUtils` : 랜덤 관련 기능
   - `ShuffleUtils` : 문제 순서 섞기, 보기 섞기
5. `index.js` : 전체 프로그램 시작점
6. `tests/` : 단위 테스트

## 예외처리 및 입력 검증

- **메뉴 번호 입력**

  - 숫자가 아닌 값 입력 → “숫자로 입력해주세요” 안내 후 재입력
  - 범위(예: 1~5)를 벗어난 숫자 → “1~5 사이 숫자를 입력해주세요” 안내 후 재입력

- **보기 번호 입력**

  - 1~4 이외 숫자나 문자를 입력하면 동일하게 재입력 요청

- **카테고리 선택**

  - 존재하지 않는 번호 선택 → 다시 선택 요청

- **데이터 파일 관련**

  - history.json이 없거나 깨져 있어도 loadHistory()에서 기본 템플릿을 자동 생성 후 사용
  - config.json이 존재한다고 가정하고 사용
